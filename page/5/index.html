1.1、python的优势和不足

优势：

  • 学习成本低，入门快
  • 开发效率快（专注于算法和解决问题，而不是语言本身）
  • 开源、跨平台，源码可读性极高（规范的代码风格，被称为可执行的伪代码）
  • 丰富的第三方库（太多了，也导致良莠不齐）

不足：

  • 性能低，运行速度慢，差c++/java 至少10倍以上
  • GIL限制，无法实现多核多线程。
  • 不适于cpu密集型。适合磁盘、网络I/O密集型
  • 开发桌面GUI程序，手机app不建议使用python
  • python程序的源码是可以被别人看到的，不适合开发某些商业产品

python性能应该是所有动态脚本语言中最快的。

3、scrapy爬虫框架的介绍和使用

3.1、介绍

好的爬虫框架至少应该把网页请求下载、数据解析、数据清洗存储分开处理。

scrapy就是这样。

为了帮助理解，先定义几个概念：

网页请求

Request：包含URL，参数，请求方法的对象。浏览器向服务器发送请求，称为request，

Response：类似Request，是服务器返回给我们的。

Downloader：处理Request对象，下载网页，生成Response对象。

内容解析

Item：我们将想要获取的信息为Item对象，比如抓新闻，Item中要定义“新闻的标题，作者，日期，新闻内容，配图”等。后面的数据处理就以Item为单位。

Spiders：解析内容的蜘蛛，从Response中提取Item对象(要抓取的信息)和Request对象(新发现的URL)

数据处理

ItemPipeline： Pipeline是管道的意思，管道里走的是Item。也就是Item经过一个管道就被处理一次(数据清洗、丢弃、存储等操作)，多个管道可以接在一起工作。

5、Scapy高级用法

5.1、分布式爬虫

分布式爬虫：解决大规模爬取数据的性能瓶颈(cpu, mem, disk,带宽)；一定程度的绕过反爬虫的能力；slaver之间的负载均能和冗余容错；

分布式的核心是通信调度，大多数情况下系统的瓶颈在Master节点。

  • 分布式架构：Master分发控制任务，Slaver负责执行任务，并反馈结果。

  • scrapy-redis插件

    redis是基于内存的no-sql数据库，可以实现消息分发订阅和url去重。

    用scrapy-redis替换scrapy框架的scheduler，redis存储Request url和crawled url。

  • 利用集群进一步提高性能

    • Master可以是单主机的redis，也可以是redis集群。
    • redis大约8G内存可以存放2亿URL，如果用布隆过滤器实现URL去重，大约512MB保存2亿url的hash信息（有1/10000的概率误判，概率可以调节）。
    • Slaver获取的数据可以存放如一个数据库，也可以是数据库集群。数据库也是可以实现分布式存储的。
    • Hadoop大数据框架好像就能实现数据的分布式存储和计算。

  • Master/Slaver之间的通信，也可以使用kafka，rabitmq等其他的消息队列技术实现。

5.2、JS脚本执行

JavaScript一种直译式脚本语言，是一种动态类型、弱类型、基于原型的语言，内置支持类型。它的解释器被称为JavaScript引擎，为浏览器的一部分，广泛用于客户端的脚本语言，最早是用来给HTML网页增加动态功能。

js脚本可以：

  • 通过程序发起Request下载下来的HTML是未经JavaScript渲染的（和浏览器查看HTMl源码得到的文件不同）

  • 很多网站为了提高数据加载效率（js异步请求数据，ajax技术）、保护登录安全（js函数加密post登录数据）、保护网站数据（js加密数据），这种情况，只能破解js函数（很难），或者渲染js脚本。

  • 爬虫实现js渲染，要么实现js引擎，要么利用webkit技术（实现一个浏览器内核）。

  • 渲染js的速度很慢，一个网页加载完成可能需要数秒。程序直接download一个html文件是很快的。

  • scrapy-splash插件，解决js的问题。

    Splash - A javascript rendering service ，可以渲染js的web浏览器，可以并行加载网页，屏蔽图片、广告，

    获取网页源码或截图…

  • splash引擎和scrapy-splash插件和scrapy框架都是一个团队开发的，因此可以放心使用。

5.3、可视化的方法编写程序

一个scrapy项目中，最耗时的部分就是页面解析规则，即css selector表达式，xpath表达式，re表达式。如果我们用鼠标在浏览器页面点一点，表达式就自动写入到程序里，就能大大提高编程效率。scrapy家的项目portia2.0提供了类似的功能。

3. 参数mode的基本取值

Character Meaning
‘r’ open for reading (default)
‘w’ open for writing, truncating the file first
‘a’ open for writing, appending to the end of the file if it exists
‘b’ binary mode
‘t’ text mode (default)
‘+’ open a disk file for updating (reading and writing)
‘U’ universal newline mode (for backwards compatibility; should not be used in new code)

r、w、a为打开文件的基本模式，对应着只读、只写、追加模式；
b、t、+、U这四个字符，与以上的文件打开模式组合使用，二进制模式，文本模式，读写模式、通用换行符，根据实际情况组合使用、

常见的mode取值组合

r或rt 默认模式，文本模式读

rb 二进制文件

w或wt 文本模式写，打开前文件存储被清空

wb 二进制写，文件存储同样被清空

a 追加模式，只能写在文件末尾

a``+ 可读写模式，写只能写在文件末尾

w``+ 可读写，与a``+``的区别是要清空文件内容

r``+ 可读写，与a``+``的区别是可以写到文件任何位置
