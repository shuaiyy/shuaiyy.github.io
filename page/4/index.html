Gevent 学习笔记

python Gevent

进程、线程、协程

python 中的yield提供对协程的有限支持。

  • 进程

    • 正在运行的程序的实例
    • 具有独立地址空间
    • 是操作系统资源分配的基本单位

    • 进程上下文： 进程的物理实体与支持进程运行的物理环境，包括地址空间，系统栈，打开文件表，……
    • 上下文切换：由一个进程的上下文转到另外一个进程的上下文
    • 系统开销：操作系统完成系统管理工作所花费的时间和空间
    • 一个进程可以包含多个线程

  • 线程

    • 线程是程序执行的最小单位
    • 多线程可以提高程序的并发性
    • 由于python的GIL机制，一个进程只能使用一个cpu核心，因此多线程并不适合解决CPU密集的运算，此时应该使用多进程。

  • 协程

    • 可以认为是一种用户态的线程
    • 线程是由系统调度，而协程需要主动让出CPU时间，即控制权在程序员手中
    • 线程里可以包含多个协程

使用pyinstaller打包PyQt5程序

打包的程序好像使用jpg和ico图片会不显示，Pixmap

安装

  • pip install pyinstaller
  • 官网文档
  • PyInstaller supports Python 2.7 and Python 3.3+

使用介绍

pyinstaller [*options*] *script* [*script* ...] | *specfile*

会生成build和dist文件夹，生成的app文件在dist下。

pyinstaller --noconfirm --log-level=WARN --onefile -c --hidden-import=sip --icon=f.ico -n="myapp" --add-data="qminimal.dll;.\a.jpg\imgs" --add-binary="qoffscreen.dll;.\dist\platforms" --add-binary="qwindows.dll;.\dist\platforms" run.py

事实上pyintsller会先根据你的参数创建specfile文件，然后打包程序。我们也可以直接编写specfile，然后pyinstaller specfile

  • myapp.spec

打包成文件夹

  • pyinstaller myscript.py 会生成包含myscript.exe及其依赖的myscript文件夹。
  • 参数-D , 默认就是这个

打包成单exe文件

  • 参数-F
  • 注意，即使打包成单个exe，其运行前也是先将文件资源解压到一个temp文件夹_MEIxxxx内,程序正常退出时会删除该文件夹。异常退出，崩溃时不会删除。
  • 单个exe启动有可能非常慢

使用/隐藏Console窗口

  • print 和 input会作用在控制台
  • 程序异常也会打印在console
  • 初次打包时应使用带console的，方便调试,检查错误

加壳和加密

  • upx下载
  • –upx-dir 指定upx的位置
  • 安装PyCrypto
  • 加密指定 –key=your_key_string

参数说明

  • --distpath DIR 指定app输出的路径
  • --workpath WORKPATH 指定build所用的文件夹
  • --upx-dir UPX_DIR 指定upx utility的位置，用于给exe加壳
  • -F –onefile 单个exe文件
  • -n NAME` 指定输出app的名字
  • --add-data FILE or DIR 添加其他文件或文件夹，多个参数使用;隔离
  • --add-binary File or Dir 同上
  • -p DIR 要导入的python 包所在的位置
  • --hidden-import MODULENAME 指定要导入的包名
  • -c 带控制台 , -w 不带控制台的窗口程序
  • -i yourico.ico 为生成的exe添加图标

打包PyQt5程序

先打包一个单文件带控制台的exe文件，然后生成的文件在cmd里打开，如果出错了可以看到错误日志。

错误解决

  1. This application failed to start because it could not find or load the Qt platform plugin "windows".

    原因： You probably need to add QT DLLs to your .spec file as datas. That error is due to Qt missing some .dll

    有说设置qt的系统变量的，但我试过没起作用。也有的是因为upx加壳造成的冲突。

    暂时没找到更好的解决办法，复制platforms文件夹和exe在同一层级，就可以正常运行了。

  2. ImportError: No module named sip. Failed to execute script run

    安装sip包，然后打包参数’–hidden-import=sip’

    python27要自行编译sip，经测试，将PyQt5.sip移动到site-packages下，可以正常运行。

  3. jpg和ico图片显示不正常

    不知道咋解决，改用原生支持的PNG格式是肯定不会有问题的。

Python super与多重继承顺序

python多重继承的顺序

Python super对象

  1. Base.__init__(self, args)

    在Python2.2以前，未绑定的父类构造方法经常被使用A.__init__(self)

    这种做法的缺点是，

    • 当多重继承以后，想要修改代码的话，比如某个基类改名字了，那么所有调用该类方法的代码部分都要修改。
    • 基类的方法可能会被多次调用
    • 要保证子类能够覆盖父类，及生成所有的子类后再生成父类。这要看代码的质量，如果继承关系复杂，很容易把人搞晕。

  2. super(Child, self).__init__(args)

    在Python类的方法中，要调用父类的某个方法，一般会用super(X, self).func(args),尤其是在类初始化中。

    使用super不会显示的调用父类方法。super对象会根据继承关系的顺序依次执行每个父类的__init__方法,如果每个类都使用super调用父类方法的话，那么可以确保每个类的__init__只会执行一次。
