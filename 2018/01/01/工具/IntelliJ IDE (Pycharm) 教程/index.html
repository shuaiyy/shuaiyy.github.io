IntelliJ IDE (Pycharm) 教程

安装与配置

  • 官网下载并安装，硬件最好8G ram + SSD

  • 激活服务器

  • JetBrains\PyCharm 2017.1.2\bin下的重要配置文件

    • idea.properties

      idea.config.path=${user.home}/.PyCharm/config可以指定配置文件的目录，多台电脑间安装pycharm可以拷贝使用同一份config。这样就只需维护一份配置，并保持所有机器同步。

    • pycharm64.exe.vmoptions IDEA 64 位的可执行文件的 VM 配置文件

      如果你是开发大型项目、Python 项目或是 Web项目，并且内存大于 8G，建议进行修改此文件。

  • 运行界面

    没有workspace的概念，最大的工作单元是project。因此查看多个项目时，可以同时打开多个project

    新建项目后，IDEA会自动创建索引，可能会卡顿，此时需稍作等待，不要乱操作。

主题、字体、编码、乱码问题

在打开project下修改settings下次关闭后在进来，某些改动会失效，可以修改default settings

Ctrl + Alt + S 快速打开设置

  • 主题

    files -> settings -> appearence -> theam下, 个人偏爱Darcula (吸血鬼德库拉伯爵) 黑色风格。改完需重启IDEA。

  • 主题字体

    files -> settings -> appearence -> Override default fonts by 下,微软雅黑和宋体是包含中文的，Monaco只有英文。

  • 代码编辑字体

    Files -> settings -> Editor -> Colors & Fonts -> Font ，默认的模板无法直接修改，要先Save As…, 编码字体有第一字体（Primary font） 和 第二字体（Secondary font）之分。当有些字符在第一字体支持不了的时候，会去使用第二字体进行支持。

    个人习惯英文字体使用 Monaco，作为第一字体，第二字体使用 Yahei Consolas Hybrid支持中文。这两个字体都不是系统自带的，需要自行下载安装。

  • 控制台输出字体，解决中文乱码

    files -> settings -> appearence -> Editor -> Colors & Fonts -> Console Font，

  • 编辑区主题

    files -> settings -> appearence -> Editor -> Colors & Fonts -> General

    常修改的有：

    • Default text，指的是默认代码文本，我一般会修改其 Background 属性。
    • Caret row，指的是光标所在行，我一般会修改其 Background 属性。
    • Vertical indent guide，指的是垂直缩进线，我一般会修改其 Foreground 属性。
    • Identifier under caret，指的是光标所在位置的相同标识符呈现什么效果，我一般会修改其 Background 属性。
    • Text search result，指的是在查找模式下，匹配字符的样式，我一般会修改其 Background 属性

  • 代码显示行号

    File –> Settings –>Editor –>Appearance , 之后勾选Show Line Numbers

  • 文件编码修改

    files -> settings -> appearence -> Editor -> Codestyle -> File Encodings

    • IDE 和Project Encoding 都建议修改为 UTF-8。
    • IntelliJ IDEA 可以对 Properties 文件进行专门的编码设置，一般也建议改为 UTF-8，其中有一个重点就是属性 Transparent native-to-ascii conversion， 主要用于转换 ascii，一般都要勾选，不然 Properties 文件中的注释显示的都不会是中文。

创建项目

  • 在项目目录上右键设为source root
  • IDEA的哲学是一个IDE窗口管理一个project，没有workspace 多project的概念，也不推荐在一个项目里创建多个project。
  • IDEA 没有ctrl + s，因此改完代码可以直接run或debug。

版本控制

首先要安装版本控制的客户端，然后在files -> settings -> Version Control 下对应的插件进行相关配置。

SVN 的配置

要在 IntelliJ IDEA 中使用 SVN，需要先安装 SVN 客户端或是 TortoiseSVN 这类图形化工具，Windows 系统这里推荐安装 TortoiseSVN，即使在不使用 IntelliJ IDEA 也可以方便管理我们的项目。

SVN 主要使用的版本有 1.6、1.7、1.8，最新的是 1.9。推荐大家使用 1.8 的。如果你的项目使用的是 1.6 的版本，在安装 1.8 之后是可以直接对项目文件进行升级的，所以无需担心，也因此更加推荐大家使用 1.8。

  • Subversion 官网下载：https://subversion.apache.org/download/#recommended-release
  • TortoiseSVN 官网下载：http://tortoisesvn.net/downloads.zh.html

Git 的配置

要在 IntelliJ IDEA 中使用 Git，需要先安装 Git 客户端，这里推荐安装官网版本。

Git 主要的版本有 1.X、2.X，最新的是 2.X，使用版本随意，但是不要太新了，不然可能 IntelliJ IDEA 小旧版本会无法支持可能。

  • Git 官网下载：http://git-scm.com/
  • TortoiseGit 官网下载：http://download.tortoisegit.org/tgit/

填写对应的账户和密码

可以Checkout项目，和分享本地项目到github上去。

VCS -> Checkout from Version Control -> Github

VCS -> Import into Version Control -> Github

Git 常见问题

  • 更新的时候报：Can’t update: no tracked branch
    • 解决办法：打开 git-bash（路径：C:\Program Files\Git\git-bash.exe），切换到这个更新不下来的项目的根目录，然后输入：git branch --set-upstream-to origin/master master，回车之后重新回到 IntelliJ IDEA 进行更新，正常就可以了。
  • 输错密码后，弹出验证的登录框没有再出现：
    • 解决办法：files -> settings -> System Settings -> Passwords 选择 Do not save, forget passwords after restart 等你确定你的密码没错后再选择保存密码方案。

代码模板

通过在File->setting->File and Code Templates设置模板代码，这样就可以在新建Python文件的时候自动带上抬头。

文件类型的预设变量请查阅官网

Debug 常用快捷键

快捷键 介绍
F7 在 Debug 模式下，进入下一步，如果当前行断点是一个方法，则进入当前方法体内，如果该方法体还有方法，则不会进入该内嵌的方法中 必备
F8 在 Debug 模式下，进入下一步，如果当前行断点是一个方法，则不进入当前方法体内 必备
F9 在 Debug 模式下，恢复程序运行，但是如果该断点下面代码还有断点则停在下一个断点上 必备
Alt + F8 在 Debug 的状态下，选中对象，弹出可输入计算表达式调试框，查看该输入内容的调试结果 必备
Ctrl + F8 在 Debug 模式下，设置光标当前行为断点，如果当前已经是断点则去掉断点
Shift + F7 在 Debug 模式下，智能步入。断点所在行上有多个方法调用，会弹出进入哪个方法
Shift + F8 在 Debug 模式下，跳出，表现出来的效果跟 F9 一样
Ctrl + Shift + F8 在 Debug 模式下，指定断点进入条件
Alt + Shift + F7 在 Debug 模式下，进入下一步，如果当前行断点是一个方法，则进入当前方法体内，如果方法体还有方法，则会进入该内嵌的方法中，依此循环进入

有时候我们可以这样粗鲁地认为 Debug 的使用就是等同于这几个快捷键的使用，所以上面的 必备 快捷键是我们必须牢记的，这些也是开发很常用的。

必备常用设置

  • 取消自动补全的大小写敏感

    Files -> settings -> Editor -> Code Completion 下Case Sensitive 改为None

  • 暂时取消语法检查

    pycharm打开较大的文件会比较卡，是因为在做代码检查。可以点击最右下角的小图标，临时将Highlight Level改为None，不检查；Syntax检查拼写错误，简单语法错误；Inspections为最高等级检查，可以检查单词拼写，语法错误，变量使用，方法之间调用等。

  • Ctrl + D

    Editor状态下， Ctrl + D 默认是用来复制所选的内容的，并黏贴在原位置之后。个人更喜欢复制所选的行数完整内容然后黏贴在新的起始行。所以进行了修改，Files -> settings -> Keymap 下，找到ctrl+d的Duplicate Line or Selection功能并remove，然后将ctrl +d 设置给Duplicate Entire Lines，提示ctrl+d还设置给了其他功能时，选择leave，即保留其他keymap选项对Ctrl+D的设置。

  • Ctrl + 空格

    默认 Ctrl + 空格 快捷键是基础代码提示、补充快捷键，但是由于我们中文系统基本这个快捷键都被输入法占用了，所以不管怎么按都是没有提示代码效果的。因此建议修改此快捷键为 Ctrl + 逗号。在keymap下搜索Basic。

  • 打开多文件时，文件名多行显示

    默认单行显示文件名，当打开的文件多了，后面的文件名会被自动折叠隐藏，因此应取消单行文件名显示。

    Files -> settings -> Editor -> Editor Tabs 下取消 Show tabs in single row

  • 忽略.idea文件夹

    Files -> settings -> Editor -> Code Style -> File Type 下ignore files and flolder 添加 .idea;.git

  • 将经常打开的文件添加到收藏夹

    选中文件，然后 Files -> Add to Favorites。在左侧窄边栏的右下部分可以看到Favorites 收藏夹组件窗口。

  • Alt + F1 + 回车 可以查看当前文件在项目中的位置

  • ctrl + 滚轮 调整字体大小

    需要在Files -> settings -> Editor -> General 下的Mouse栏勾选Change font size with Ctrl+Mouse Wheel

  • 设置显示代码行号和方法分隔线

    Files -> settings -> Editor -> Appearance下，方法分割线看个人喜好设置。

  • ctrl alt T 选中想要折叠的代码块进行折叠，并添加注释

  • 编辑某个文件时，project导航窗自动定位到文件所在的位置

    在project导航窗口的右上角。点击齿轮图标，选中 Autoscroll from Source

  • 本地历史记录 源码文件编辑窗右键，选择Local History。文件夹也可以查看历史。

  • RESTful API 测试工具 在Tools下

  • 快速注释 Ctrl + / /也叫Slash

  • TODO工具窗

    可以在要实现或修改的代码部分添加todo标记，下次就可以快速定位到该位置。标记可以自己定义，点击矿口左侧的漏斗图标，自定义过滤器。默认定义了\btodo\b.* 和\bfixme\b.*

    | filter pattern | comment string |
    | ————– | ————————– |
    | \btodo\b. | #todo 此处应实现功能XXX |
    | \bfixme\b.     | #fixme 此处要修改XXX |
    | \b未完成\b. | 只要匹配该规则的注释内容，都会被TODO工具定位到。 |
    | \b要修改\b.     | |

  • DEBUG窗口有一个批量删除断点的按钮

常用快捷键

关闭其他软件的快捷键

QQ、搜狗输入法、Snipaste的快捷键都关掉，以免冲突。

  • 搜狗 输入法管理器中设置搜狗为默认中文输入法，快捷键为ctrl + ~, 然后按键栏的系统功能快捷键和扩展功能快捷键全部取消
  • QQ热键仅保留截图
  • 网易音乐 有道词典等的快捷键

快捷键 介绍
